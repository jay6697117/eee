# 项目缺口检测

扫描项目状态，找出代码和文档之间的差距。

## 步骤

// turbo
1. 运行缺口检测脚本：
```bash
bash .agent/hooks/detect-gaps.sh
```

2. 分析输出结果，可能的检测项包括：

   - 🚀 **新项目检测** — 如果没有引擎配置、没有游戏概念、没有源码，建议运行 `/start`
   - ⚠️ **代码多但设计文档少** — 建议运行 `/reverse-document`
   - ⚠️ **原型无文档** — 原型目录缺少 README 或 CONCEPT 文件
   - ⚠️ **核心系统无架构文档** — 建议运行 `/architecture-decision`
   - ⚠️ **Gameplay 系统无设计文档** — 建议运行 `/reverse-document`
   - ⚠️ **大代码库无生产计划** — 建议运行 `/sprint-plan`

3. 将检测结果整理成清晰的报告呈现给用户，并附上每个问题的建议修复命令。
