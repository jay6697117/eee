# 推送前检查

在执行 `git push` 之前运行此命令，避免误推到受保护分支。

## 步骤

// turbo
1. 检查当前分支：
```bash
git rev-parse --abbrev-ref HEAD
```

2. 判断当前分支是否是受保护分支（`main`、`master`、`develop`）。

3. 如果**是受保护分支**，向用户发出警告：
   > ⚠️ 你正在 `[分支名]` 上，这是一个受保护分支。
   > 推送前请确认：
   > - ✅ 构建通过
   > - ✅ 单元测试通过
   > - ✅ 无 S1/S2 级别 Bug
   > 
   > 确认要继续推送吗？

4. 如果**不是受保护分支**，报告安全状态：
   > ✅ 当前分支 `[分支名]` 不是受保护分支，可以安全推送。
