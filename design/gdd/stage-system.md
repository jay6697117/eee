# 场景/舞台系统 (Stage System)

> **状态**: 已设计
> **最后更新**: 2026-03-25
> **服务支柱**: 视觉轰炸 / 个性鲜明

## 概述

管理战斗场景的背景渲染、边界定义和场景专属 BGM。每个角色有一个专属主场场景，MVP 阶段只需 1 个场景。

## 详细设计

### 场景配置

```javascript
{
  id: "colosseum",
  name: "时空竞技场",
  bgLayers: [                    // 视差滚动背景层
    { image: "bg_sky.png", scrollFactor: 0.1 },     // 远景（天空）
    { image: "bg_pillars.png", scrollFactor: 0.3 },  // 中景（柱子）
    { image: "bg_floor.png", scrollFactor: 1.0 },    // 近景（地面）
  ],
  groundY: 600,                  // 地面 Y 坐标
  stageWidth: 1280,              // 舞台宽度
  boundaryLeft: 50,              // 左边界
  boundaryRight: 1230,           // 右边界
  bgm: "audio/bgm_colosseum.mp3",
  ambientSfx: "audio/sfx_crowd.mp3",  // 环境音
}
```

### MVP 场景：时空竞技场

- **视觉**：古罗马竞技场 + 时空裂缝元素，柱子间有蓝紫色能量涌动
- **层次**：3 层视差背景（天空/建筑/地面）
- **动态**：背景观众轻微动画，火把闪烁
- **BGM**：激昂的战斗音乐

### 与其他系统交互

| 系统 | 数据流向 | 说明 |
|------|----------|------|
| 本系统 → 物理/碰撞 | 输出 | 舞台边界和地面 Y |
| 对战模式 → 本系统 | 输入 | 选择哪个场景 |
| 菜单/选人 → 本系统 | 查询 | 场景预览 |

## 验收标准

- [ ] 背景正确渲染（3 层视差）
- [ ] 舞台边界约束角色活动范围
- [ ] BGM 循环播放
- [ ] 场景切换时正确加载资源
