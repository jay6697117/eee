# 动画系统 (Animation System)

> **状态**: 已设计
> **最后更新**: 2026-03-25
> **服务支柱**: 视觉轰炸 / 个性鲜明

## 概述

管理角色精灵帧动画的播放、切换和事件回调。每个动画帧绑定碰撞箱数据（hitBox/hurtBox），为战斗系统提供逐帧精确的判定信息。

### 帧数据结构

```javascript
// 每一帧包含的数据
{
  frameIndex: 0,        // 精灵帧索引
  duration: 1,          // 持续游戏帧数
  hurtBoxes: [          // 本帧可被攻击区域
    { x: -20, y: -80, w: 40, h: 80 }
  ],
  hitBoxes: [           // 本帧攻击判定（仅攻击动画有）
    { x: 30, y: -50, w: 50, h: 30, damage: 50, hitstun: 10, knockback: 5 }
  ],
  pushBox: { x: -18, y: -80, w: 36, h: 80 },
  events: ['playSound:punch'],  // 帧事件
  cancelable: false,    // 是否可取消到其他招式
}
```

### 动画事件

动画帧可触发事件回调：

| 事件 | 说明 |
|------|------|
| `playSound:name` | 播放音效 |
| `spawnEffect:name` | 生成粒子特效 |
| `screenShake:intensity` | 屏幕震动 |
| `hitStop:frames` | 打击停顿 |
| `cancelWindow:start` | 开始取消窗口 |
| `cancelWindow:end` | 结束取消窗口 |
| `moveForward:px` | 前进位移 |
| `invincible:start` | 无敌开始 |
| `invincible:end` | 无敌结束 |

### 与其他系统交互

| 系统 | 数据流向 | 说明 |
|------|----------|------|
| 角色状态 → 本系统 | 输入 | 状态变化触发动画切换 |
| 本系统 → 物理/碰撞 | 输出 | 每帧的碰撞箱数据 |
| 本系统 → 连招系统 | 输出 | 取消窗口事件 |
| 本系统 → 战斗系统 | 输出 | hitBox 数据和伤害属性 |

## 调优参数

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| `ANIM_SPEED_SCALE` | 1.0 | 0.5-2.0 | 全局动画速度倍率 |
| `HIT_STOP_FRAMES` | 4 | 2-8 | 默认打击停顿帧数 |
| `SCREEN_SHAKE_DECAY` | 0.9 | 0.8-0.95 | 震屏衰减速率 |

## 验收标准

- [ ] 动画状态切换无跳帧
- [ ] 每帧碰撞箱精确对应精灵位置
- [ ] 帧事件按时触发
- [ ] 打击停顿效果明显
- [ ] 非循环动画播完后回调正确
