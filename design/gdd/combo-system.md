# 连招系统 (Combo System)

> **状态**: 已设计
> **最后更新**: 2026-03-25
> **服务支柱**: 上手即爽 / 视觉轰炸

## 概述

连招系统管理攻击链的衔接逻辑，包括取消窗口（Cancel Window）、连招路线表和连击计数器。让新手按按钮就能打出好看的连招，让高手通过精确时机最大化伤害。

## 详细设计

### 连招路线（Chain Route）

```
基础连招链（所有角色通用）：
轻攻击 → 轻攻击 → 重攻击                   // 基础 3 连
轻攻击 → 轻攻击 → 必杀技                   // 必杀收尾
轻攻击 → 重攻击 → 必杀技                   // 高伤害路线
蹲轻攻击 → 蹲重攻击 → 必杀技               // 下段路线
空中攻击 → (着地) → 轻攻击 → ...           // 空对地接续
任意攻击 → (觉醒中) → 超必杀               // 觉醒终结

取消优先级：轻 → 重 → 必杀 → 超必杀（单向递增）
```

### 取消窗口

```
每个攻击动画有一个取消窗口区间 [startFrame, endFrame]：
- 轻攻击: 命中帧 ~ 命中帧+4
- 重攻击: 命中帧 ~ 命中帧+3
- 必杀技: 命中帧 ~ 命中帧+2

取消条件:
  1. 当前帧在取消窗口内
  2. 本次攻击命中了对手（空振不可取消）
  3. 目标招式优先级 > 当前招式
  4. 输入缓冲中有对应指令
```

### 连击计数器

```
comboCount: 当前连击数
comboTimer: 连击超时计时器

命中时:
  comboCount++
  comboTimer = COMBO_TIMEOUT  // 重置超时

每帧:
  if (comboTimer > 0) comboTimer--
  if (comboTimer == 0) comboCount = 0  // 连击中断

显示: 连击数 ≥ 2 时在屏幕显示 "X HITS!"
```

### 与其他系统交互

| 系统 | 数据流向 | 说明 |
|------|----------|------|
| 输入系统 → 本系统 | 输入 | 按键序列 |
| 战斗系统 → 本系统 | 事件 | 命中事件触发连击计数 |
| 动画系统 → 本系统 | 数据 | 取消窗口帧信息 |
| 本系统 → 战斗系统 | 查询 | 提供 comboCount 用于伤害递减 |
| 本系统 → 战斗 UI | 输出 | 连击数显示 |

## 调优参数

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| `COMBO_TIMEOUT` | 30 | 20-45 | 连击超时（帧） |
| `CANCEL_WINDOW_LIGHT` | 4 | 3-6 | 轻攻击取消窗口 |
| `CANCEL_WINDOW_HEAVY` | 3 | 2-5 | 重攻击取消窗口 |
| `CANCEL_WINDOW_SPECIAL` | 2 | 1-3 | 必杀技取消窗口 |

## 验收标准

- [ ] 轻→轻→重 基础连招流畅衔接
- [ ] 空振不可取消
- [ ] 连击计数正确显示
- [ ] 连击中断后计数归零
- [ ] 取消窗口时机精确
