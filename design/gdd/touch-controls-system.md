# 触屏控制 UI 系统 (Touch Controls System)

> **状态**: 已设计
> **最后更新**: 2026-03-25
> **服务支柱**: 上手即爽

## 概述

为移动端提供虚拟摇杆和攻击按钮。仅在触屏设备上显示，PC 端隐藏。采用动态定位（触屏位置出现摇杆）的方式提升操控舒适度。

## 详细设计

### 布局

```
┌──────────────────────────────────────────────┐
│                                              │
│                   战斗区域                    │
│                                              │
│                                              │
│   ┌─────┐                       [C]         │
│   │     │                    [A]   [B]       │
│   │  ◉  │  ← 动态虚拟摇杆                    │
│   │     │                    [防御]           │
│   └─────┘                                    │
└──────────────────────────────────────────────┘
  左半屏:移动                右半屏:攻击
```

### 虚拟摇杆（左半屏）

```
触发: 左半屏任意位置按下
显示: 按下位置出现摇杆底盘（120px直径）
操作: 
  - 拖动 → 移动方向
  - 上滑 → 跳跃
  - 下拖 → 蹲下
  - 释放 → 回中，停止移动
  
死区: 中心 15px（防误触）
最大距离: 50px（超出则钳制）
```

### 攻击按钮（右半屏）

| 按钮 | 大小 | 位置 | 功能 |
|------|------|------|------|
| A（轻攻击） | 60px | 右下偏左 | 轻攻击 |
| B（重攻击） | 60px | 右下偏右 | 重攻击 |
| C（元素技能） | 50px | A+B 上方 | 元素技能 |
| 防御 | 50px | A+B 下方 | 防御 |

### 必杀技触发

```
必杀技 1: A+B 同时按（两指同时点）
超必杀: A+B+C 同时按（三指同时点）
必杀技 2: 摇杆↓→ + C（方向组合，与键盘逻辑统一）
```

### 移动端适配

```
检测设备:
  if ('ontouchstart' in window || navigator.maxTouchPoints > 0)
    → 显示触屏控件
    → 隐藏 P2 键盘映射提示
  else
    → 隐藏触屏控件
```

### 视觉样式

```
透明度: 0.4（不遮挡战斗）
按下时: 透明度 → 0.8 + 轻微缩放
颜色: 半透明白色描边 + 深色填充
按钮图标: 简单字母（A/B/C）
```

### 与其他系统交互

| 系统 | 数据流向 | 说明 |
|------|----------|------|
| 本系统 → 输入系统 | 输出 | 将触屏事件转换为统一游戏指令 |

## 调优参数

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| `JOYSTICK_SIZE` | 120 | 80-160 | 摇杆底盘直径 |
| `JOYSTICK_DEADZONE` | 15 | 10-20 | 死区半径 |
| `BUTTON_SIZE` | 60 | 40-80 | 攻击按钮直径 |
| `TOUCH_OPACITY` | 0.4 | 0.2-0.6 | 默认透明度 |
| `TOUCH_ACTIVE_OPACITY` | 0.8 | 0.6-1.0 | 按下时透明度 |
