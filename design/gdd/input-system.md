# 输入系统 (Input System)

> **状态**: 已设计
> **最后更新**: 2026-03-25
> **服务支柱**: 上手即爽

## 概述

输入系统统一处理键盘和触屏输入，将物理输入转换为游戏指令（移动、攻击、防御、技能）。支持双人本地对战的输入分区和方向组合识别（用于简化必杀技指令）。

## 玩家体验

玩家感受到的是"操作跟手" — 按下按钮瞬间角色就做出反应。简化指令让新手 30 秒上手，输入缓冲让连招流畅不卡顿。

## 详细设计

### 输入映射

#### 玩家 1（键盘左区）

| 动作 | 按键 |
|------|------|
| 左移 | A |
| 右移 | D |
| 跳跃 | W |
| 蹲下 | S |
| 轻攻击 | J |
| 重攻击 | K |
| 元素技能 | L |
| 防御 | U |

#### 玩家 2（键盘右区）

| 动作 | 按键 |
|------|------|
| 左移 | ← |
| 右移 | → |
| 跳跃 | ↑ |
| 蹲下 | ↓ |
| 轻攻击 | Numpad 1 |
| 重攻击 | Numpad 2 |
| 元素技能 | Numpad 3 |
| 防御 | Numpad 0 |

#### 触屏（手机端，由触屏控制 UI 系统提供）

| 动作 | 控件 |
|------|------|
| 移动 | 虚拟摇杆（左半屏） |
| 跳跃 | 上滑 / 摇杆上 |
| 蹲下 | 下滑 / 摇杆下 |
| 轻攻击 | A 按钮 |
| 重攻击 | B 按钮 |
| 元素技能 | C 按钮 |
| 防御 | 双指同时按 |

### 必杀技指令（简化型）

| 招式 | 指令 | 输入窗口 |
|------|------|----------|
| 必杀技 1 | → + A+B | 同时按下（±3帧容差） |
| 必杀技 2 | ↓→ + C | 方向组合后 8 帧内按 C |
| 超必杀技 | A+B+C | 三键同时（±3帧容差），需怒气满 |

### 输入缓冲

```
缓冲窗口：6 帧（100ms @ 60fps）
机制：在当前动作结束前 6 帧内的输入会被缓存，
      动作结束后立即执行缓冲中的输入。
作用：让连招衔接更流畅，降低操作门槛。
```

### 方向组合检测

```
检测窗口：12 帧（200ms）
队列长度：4（最多记录最近 4 个方向输入）
例：↓→ + C
  1. 检测到 ↓ 按下，记入方向队列
  2. 12 帧内检测到 → 按下，记入队列
  3. 检测到 C 按下，检查方向队列是否匹配 ↓→
  4. 匹配成功 → 触发必杀技 2
```

### 与其他系统交互

| 系统 | 数据流向 | 说明 |
|------|----------|------|
| 本系统 → 角色状态 | 输出 | 发送游戏指令驱动状态转换 |
| 本系统 → 连招系统 | 输出 | 发送输入序列用于连招判定 |
| 触屏控制 UI → 本系统 | 输入 | 触屏虚拟按键转换为统一指令 |

## 调优参数

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| `INPUT_BUFFER_FRAMES` | 6 | 3-10 | 输入缓冲窗口（帧） |
| `DIRECTION_COMBO_WINDOW` | 12 | 8-16 | 方向组合检测窗口 |
| `SIMULTANEOUS_TOLERANCE` | 3 | 2-5 | 同时按键容差（帧） |
| `DIRECTION_QUEUE_SIZE` | 4 | 3-6 | 方向队列长度 |

## 边界情况

| 情况 | 处理 |
|------|------|
| 键盘和触屏同时输入 | 两者合并，不冲突 |
| 对角按键（↓+→同时） | 取最后按下的方向 |
| 暂停状态输入 | 忽略所有游戏输入，仅响应暂停菜单 |
| 浏览器失焦 | 释放所有按键状态，防止卡键 |

## 验收标准

- [ ] 键盘输入延迟 < 16ms（1 帧）
- [ ] 输入缓冲正常工作（连招衔接流畅）
- [ ] 方向组合准确识别 ↓→ + 按键
- [ ] P1/P2 输入互不干扰
- [ ] 浏览器失焦后无卡键
- [ ] 触屏输入和键盘输入统一接口
